# SellerOS V1 Launch Backlog

## Must Ship

- provider onboarding with manual form flow
- provider onboarding with OpenAPI import flow
- hosted credential mode
- self-hosted relay mode
- listed and unlisted service support
- payable service endpoint with 402 quote path
- X Layer payment verification
- service card generation
- agent-readable directory API
- seller dashboard with income, calls, latency, success rate, and recent transactions

## Should Ship

- credential rotation UI
- direct copy button for payable endpoint
- service disable and republish controls
- seeded demo data
- concise operator logs for failed payments and failed fulfillments

## Can Wait

- non-zero platform fee
- policy templates wired to enforcement
- recommendation logic beyond sort and filter
- richer analytics windows
- provider team roles and permissions
- comment or review systems

## Explicitly Not V1

- multi-chain payments
- package pricing and subscriptions
- escrow, arbitration, or automatic refunds
- advanced AI routing across multiple providers
- full reputation protocol
