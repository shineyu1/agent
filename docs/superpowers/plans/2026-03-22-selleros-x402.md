# SellerOS for x402 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the first operational version of SellerOS for x402 so a provider can self-onboard an existing API, publish a payable service, accept one paid request on X Layer, fulfill it, and review the result in a dashboard.

**Architecture:** Use a single Next.js TypeScript app for the first release so UI, directory, provider console, and gateway logic ship together. Keep responsibilities separated in focused modules: service registry, upstream connector, quote/payment gateway, directory API, and analytics.

**Tech Stack:** Next.js, TypeScript, Tailwind CSS, Postgres, Prisma ORM, x402 / payment integration SDK, zod, React Hook Form, NextAuth or wallet-first auth, Vitest, Playwright

---

## File Structure

Create these top-level units:

- `package.json` for workspace scripts
- `app/` for routes, pages, and API endpoints
- `components/` for UI components grouped by feature
- `lib/db/` for Prisma client and schema helpers
- `lib/services/registry/` for service definitions and onboarding logic
- `lib/services/gateway/` for quote generation, payment verification, and fulfillment
- `lib/services/discovery/` for service cards and directory search
- `lib/services/analytics/` for dashboard aggregation and reputation metrics
- `lib/services/security/` for credential storage helpers
- `prisma/schema.prisma` for database models
- `tests/` for unit and end-to-end coverage
- `docs/` for product and implementation docs
